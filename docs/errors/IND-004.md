# IND-004: Regime optimizer returned a degenerate best-trial

## Why this error fires

Every Optuna trial violated at least one hard constraint (min_ranging_pct / min_trending_pct / etc.), so the best-trial is the first-evaluated arbitrary trial, not a validated result. Deploying these params is unsafe.

## Fix

Loosen constraints in RegimeOptimizerConfig, or increase the historical window so the optimizer has enough regime-segments to satisfy constraints:
  n_trials:          {n_trials}
  trials_rejected:   {trials_rejected}
  rejected_reasons:  {rejected_reasons}
