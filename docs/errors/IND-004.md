# IND-004: Regime optimizer produced a degenerate result

## Why this error fires

`_build_summary_result` (in
`echolon/indicators/optimization/interday_regime_optimizer.py`) marks the
optimizer output as **degenerate** when every trial was rejected by
constraint checks — i.e. `len(degenerate_trials) == n_trials and n_trials > 0`.

In that case the returned summary dict carries:

```python
{
    "best_params": <whatever Optuna picked, scored 0.0>,
    "n_trials": 100,
    "trials_rejected": 100,
    "degenerate": True,
    "code": "IND-004",
    "rejected_reasons": {"min_trending_pct": 62, "min_volatile_pct": 38},
}
```

and the module logs a WARNING pointing at the per-constraint rejection
counts. IND-004 is the **aggregate** counterpart to
[BT-003](BT-003.md), which fires per-rejected-trial.

IND-004 is surfaced as a log-level diagnostic — not raised. Callers that
consume the summary dict should check `summary.get("code") == "IND-004"`
and refuse to use `best_params`: all trials failing means the "best" was
score-0 noise.

## Fix

Widen whichever constraint dominates `rejected_reasons`, or widen the
parameter ranges so the trials can produce regime splits that satisfy the
constraints.

Practical order of operations:

1. Read `rejected_reasons` in the summary dict.
2. Widen the single loudest constraint first
   (e.g. if `min_trending_pct` has 62/100 rejections, lower it from 5% to 2%).
3. Re-run. If `degenerate` is still True, move to the next loudest.

## Example

Triggering code (constraints tighter than realistic markets produce):

```python
from echolon.indicators.optimization.interday_regime_optimizer import (
    RegimeOptimizerConfig,
    optimize_regime_params,
)

cfg = RegimeOptimizerConfig(
    n_trials=100,
    min_trending_pct=30.0,   # demand 30% trending in each direction
    min_volatile_pct=20.0,   # plus 20% volatile
    max_single_regime_pct=30.0,  # leaves only 20% slack — no split satisfies
)
summary = optimize_regime_params(ctx, data_dir=..., config=cfg)
# summary["code"] == "IND-004"
# summary["rejected_reasons"] shows which constraints dominated
```

Corrected (defaults, which match realistic SHFE distributions):

```python
cfg = RegimeOptimizerConfig(
    n_trials=100,
    min_trending_pct=2.0,
    min_volatile_pct=1.0,
    max_single_regime_pct=85.0,
)
summary = optimize_regime_params(ctx, data_dir=..., config=cfg)
assert not summary.get("degenerate"), summary.get("rejected_reasons")
```

Caller pattern when consuming the summary downstream:

```python
if summary.get("code") == "IND-004":
    raise RuntimeError(
        f"Optimizer returned degenerate params: {summary['rejected_reasons']}"
    )
best_params = summary["best_params"]
```
