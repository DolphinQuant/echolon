# VAL-003: Component class signature mismatch

## What
A component class (`entry_rule`, `exit_rule`, `risk_manager`, `position_sizer`) has an `__init__` signature that doesn't match the required shape.

## Why
`StrategyLoader` instantiates every component with `ComponentClass(trading_engine, **params)`. If the `__init__` signature is different, instantiation fails or silently drops the params dict.

Every component must also call `super().__init__(trading_engine, **params)` so `BaseComponent` can wire up `self.params`, `self.portfolio`, and `self.trading_engine`.

## Fix
Use the exact required signature:

```python
from echolon.quant_engine.core.base.base_component import BaseComponent


class entry_rule(BaseComponent):
    def __init__(self, trading_engine, **params):
        super().__init__(trading_engine, **params)
        # store your own params below
        self.rsi_period = self.params["rsi_period"]
```

Do not:
- Rename the first arg (`self, engine, ...`) — works, but convention is `trading_engine`
- Accept a single `params` dict (`def __init__(self, trading_engine, params):`)
- Forget the `super().__init__` call

## Common Causes
- Refactoring across Python class styles (dataclass, attrs, plain)
- LLM agent emitting `def __init__(self, **config):`
- Copy-paste from a non-Echolon codebase
