# VAL-003: Required JSON key missing from expected artifact

## Why this error fires

`validate_debug_completion` in `echolon/strategy/validators/validate_debug_completion.py`
raises VAL-003 when a validator reads a JSON artifact file (e.g., 
`selected_robust_trial.json`, `regime_metrics.json`) and finds the file exists
but lacks one or more expected top-level keys. The artifact may be corrupt,
truncated, or produced by a different pipeline stage than expected.

The context dict on the exception carries:

- `file` — the path to the JSON file that was checked
- `missing_keys` — list of expected keys that were not found
- `present_keys` — list of keys actually in the file

## Fix

Inspect the artifact file and verify that the producing stage ran to completion.
If the file is genuinely truncated or corrupted, delete it and re-run the
stage that generates it. If the file structure is intentional (e.g., a 
simplified artifact), update the validator's expected-keys list to match.

Common causes:

- **Incomplete stage execution**: The stage that writes the JSON exited early.
  Check its logs for exceptions.
- **Wrong artifact format**: The stage that wrote the JSON is from a different
  version or pipeline variant. Verify pipeline alignment.
- **File write race**: The artifact was being written when the validator read it.
  Re-run the validator.

## Example

Artifact file that triggers the error:

```json
{
  "trial_number": 42,
  "params": { "cci_period": 14, "cci_threshold": 100 }
}
```

Validator expects but does not find:

```python
expected_keys = ["trial_number", "params", "metrics"]
# Missing: "metrics"
```

Fix: Either add the missing key to the JSON, or update the validator to not
require it:

```json
{
  "trial_number": 42,
  "params": { "cci_period": 14, "cci_threshold": 100 },
  "metrics": { "sharpe": 1.25, "annual_return": 0.15 }
}
```
