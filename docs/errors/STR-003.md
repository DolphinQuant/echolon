# STR-003: Required method not implemented

## What
A component class exists with the right name, but does not implement the abstract method its base class requires.

## Why
Every `BaseComponent` subclass must implement exactly one domain method:

| Class | Required method |
|-------|-----------------|
| `entry_rule` | `generate_signal(self) -> EntrySignalOutput` |
| `exit_rule` | `should_exit(self) -> ExitSignalOutput` |
| `risk_manager` | `can_trade(self) -> RiskOutput` |
| `position_sizer` | `calculate_size(self, signal_data) -> SizerOutput` |

The strategy coordinator (`strategy_main`) must implement `_execute_bar(self) -> None` — NOT `on_bar`.

## Fix
Add the missing method. Example for `entry_rule`:

```python
class entry_rule(BaseComponent):
    def __init__(self, trading_engine, **params):
        super().__init__(trading_engine, **params)

    def generate_signal(self) -> EntrySignalOutput:
        regime = self.get_market_regime()
        return EntrySignalOutput(
            signal="HOLD", strength=0.0, type="hold",
            entry_reason="Not implemented yet", regime=regime,
        )
```

## Common Causes
- Overriding `on_bar` in `strategy_main` instead of `_execute_bar` (hooks won't fire)
- LLM emitting `generate_entry_signal` or `entry()` instead of `generate_signal`
- Forgetting to rename a method after copying from a different framework
