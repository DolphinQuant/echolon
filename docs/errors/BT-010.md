# BT-010: Required log marker absent from backtest output

## Why this error fires

`validate_debug_completion` in `echolon/strategy/validators/validate_debug_completion.py`
raises BT-010 when a debug-completion check scans the backtest log and does not
find an expected marker substring (e.g., "STAGE 4 COMPLETE", "FINAL SUCCESS").
The marker indicates that a producing stage ran to completion. Its absence
suggests the stage either failed partway through or was skipped.

The context dict on the exception carries:

- `log_path` — the path to the log file that was scanned
- `missing_marker` — the marker substring that was expected
- `last_marker_seen` — the last marker that was found before reaching EOF

## Fix

Inspect the log file around the last completed stage. Look for exception traces,
early exits, or incomplete output. Common causes:

- **Stage crashed**: The stage that produces the marker exited with an error.
  Check the traceback in the log.
- **Stage skipped**: The stage was conditional and the condition was not met.
  Verify pipeline configuration.
- **Incomplete write**: The stage was killed or timed out before writing its
  completion marker. Increase timeout or investigate resource constraints.

If the log is complete and the marker was never written, add the marker to the
stage's code before the final return or exit.

## Example

Log file that triggers the error:

```
2026-04-24 10:15:32 INFO  STAGE 1 STARTED
2026-04-24 10:15:40 INFO  STAGE 1 COMPLETE
2026-04-24 10:15:41 INFO  STAGE 2 STARTED
2026-04-24 10:15:50 INFO  STAGE 2 COMPLETE
2026-04-24 10:15:51 INFO  STAGE 3 STARTED
2026-04-24 10:16:05 ERROR Exception in optimizer: ZeroDivisionError
2026-04-24 10:16:05 ERROR Exiting
```

Validator expecting "STAGE 4 COMPLETE" but finding "STAGE 2 COMPLETE" as the
last marker triggers BT-010.

Fix: Either debug why STAGE 3 failed to complete, or lower the expected marker
to a stage that actually completed. If STAGE 3 is supposed to finish, inspect
the ZeroDivisionError and fix the underlying bug.
