# PRM-001: Missing 'printlog' key in component params

## What
`strategy_params.py` defines `DEFAULT_PARAMS`, but one of the component sub-dicts (`entry_params`, `exit_params`, `risk_params`, `sizer_params`) is missing the required `'printlog'` key.

## Why
Every `BaseComponent` reads `self.params["printlog"]` to decide whether to emit structured log lines. If it's absent, the base class raises a `KeyError` during initialization — before any bar is processed.

Keeping `printlog` explicit per-component lets you enable verbose logging on one part of the strategy while the rest stays quiet.

## Fix
Add `'printlog': False` to each component's params dict:

```python
DEFAULT_PARAMS = {
    "entry_params": {
        "printlog": False,
        "rsi_period": 14,
    },
    "exit_params": {
        "printlog": False,
        "bars_held_max": 10,
    },
    "risk_params": {
        "printlog": False,
        "max_daily_loss_pct": 2.0,
    },
    "sizer_params": {
        "printlog": False,
        "risk_per_trade_pct": 1.0,
    },
}
```

Set to `True` during debugging for the component you care about.

## Common Causes
- Writing `strategy_params.py` from scratch instead of copying the template
- Pruning "unused" keys during a refactor
- LLM agent collapsing boilerplate
