# IND-005: Calculator received a DataFrame without a required column

## Why this error fires

Indicator calculators have explicit column contracts (e.g., a session-phase indicator requires 'datetime' and 'trading_date'). Running the calculator on a DataFrame missing those columns silently produces all-NaN output in the best case, junk values in the worst.

## Fix

Ensure the input DataFrame has all required columns before calling the calculator:

  missing_column:     {missing_column}
  required_columns:   {required_columns}
  present_columns:    {present_columns}
