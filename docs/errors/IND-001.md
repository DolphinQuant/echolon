# IND-001: Indicator name casing mismatch between code and JSON

## What
Your strategy code calls `self.get_indicator("ATR_14")` (uppercase), but the pre-computed indicator column is named `atr_14` (lowercase).

## Why
Echolon pre-computes indicators into a DataFrame where all column names are lowercase. `get_indicator` does a literal column lookup — no case folding, no fuzzy matching. An uppercase call produces either a silent `KeyError` at runtime or returns `NaN`, both of which can corrupt an entire backtest before you notice.

## Fix
Lowercase every indicator name in code:

```python
# wrong
atr = self.get_indicator("ATR_14")
rsi = self.get_indicator(f"RSI_{self.rsi_period}")

# right
atr = self.get_indicator("atr_14")
rsi = self.get_indicator(f"rsi_{self.rsi_period}")
```

`strategy_indicator_list.json` declares indicators by uppercase family name (`"ATR"`) with numeric parameters — that's the DEFINITION format. The COLUMN NAME produced from that definition is always lowercase: `atr_<period>`.

## Common Causes
- Following the JSON's uppercase convention into Python
- Auto-complete from pandas / NumPy docs that use uppercase
- LLM agent following TradingView / Pine Script conventions
