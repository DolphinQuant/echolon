# PRM-002: Strategy params structure mismatch

## What
`strategy_params.py` does not export `DEFAULT_PARAMS`, or `DEFAULT_PARAMS` is missing one of the required top-level keys.

## Why
Echolon distributes parameters to components by key. The loader does:

```python
entry = entry_rule(engine, **DEFAULT_PARAMS["entry_params"])
exit_ = exit_rule(engine, **DEFAULT_PARAMS["exit_params"])
risk = risk_manager(engine, **DEFAULT_PARAMS["risk_params"])
sizer = position_sizer(engine, **DEFAULT_PARAMS["sizer_params"])
```

All four sub-dicts must exist, even if a component has no tunable parameters.

## Fix
Define `DEFAULT_PARAMS` with all four sub-dicts at module top level:

```python
# strategy_params.py
DEFAULT_PARAMS = {
    "entry_params": {
        "printlog": False,
        "rsi_period": 14,
    },
    "exit_params": {
        "printlog": False,
    },
    "risk_params": {
        "printlog": False,
    },
    "sizer_params": {
        "printlog": False,
        "risk_per_trade_pct": 1.0,
    },
}
```

If you need Optuna search spaces, also export `PARAMETER_SPECS` (see any bundled template for the shape).

## Common Causes
- Using a different top-level variable name (`PARAMS`, `default_params`)
- Nesting params under a `"strategy"` key
- Using a dataclass or BaseModel instead of a dict
