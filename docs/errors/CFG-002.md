# CFG-002: Required directory does not exist

## What
One of the paths in `BacktestConfig` (`strategy_dir`, `market_data_dir`, `indicator_dir`, or `results_dir`) points to a directory that does not exist on disk.

## Why
Echolon cannot read from a path that doesn't exist. The validator fails fast so you don't get a cryptic `FileNotFoundError` deep inside the data loader.

## Fix
Either create the directory or correct the path:

```bash
mkdir -p ./data/market ./data/indicators ./results
```

Or update your config to point at an existing directory:

```python
from pathlib import Path
cfg = BacktestConfig(
    start_date="2020-01-01",
    end_date="2023-12-31",
    strategy_dir=Path("./my_strategy"),
    market_data_dir=Path("/srv/data/echolon/market"),
    indicator_dir=Path("/srv/data/echolon/indicators"),
    results_dir=Path("./results"),
)
```

`results_dir` is the only one Echolon will create for you if missing — the others must exist before the run.

## Common Causes
- Running from the wrong working directory (relative path resolves elsewhere)
- Using `~` without `Path.expanduser()` — `Path("~/data")` does NOT expand
- Copy-pasting a config from a different machine
- Forgetting to clone the data repo / run the data pipeline
