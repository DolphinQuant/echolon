# PRM-004: Defensive .get() on self.params

## Why this error fires

`validate_parameter_access` in `echolon/strategy/validators/validate_parameter_access.py`
raises PRM-004 when it detects code using `self.params.get('x', default)`.
This pattern masks missing-parameter bugs by silently falling back to a
default instead of failing fast. The framework contract is clear: every
parameter declared in `strategy_params.py` is present on `self` as a direct
attribute, accessed via `self.x`. A missing parameter should be a load-time
error, not a runtime fallback.

The context dict on the exception carries:

- `file` — the source file containing the .get() call
- `line` — the line number
- `call` — the full .get() expression

## Fix

Replace `self.params.get('x', default)` with direct attribute access `self.x`.
This makes missing parameters fail loudly at load time, catching bugs early.
The framework guarantees that any parameter in `strategy_params.py` is injected
into the component as an attribute.

If a parameter is truly optional (not in `strategy_params.py`), the correct
approach is to:

1. Add it to `strategy_params.py` with a sensible default
2. Access it via `self.<param>`

Do not use .get() as a workaround for optional parameters.

## Example

Component using defensive .get():

```python
class EntryRule(BaseComponent):
    def generate_signal(self, bar_index: int) -> EntrySignalOutput:
        # WRONG: uses .get() to hide missing parameters
        cci_threshold = self.params.get('cci_threshold', 100)
        cci_value = self.data[bar_index, 'cci']
        if cci_value < cci_threshold:
            return EntrySignalOutput(signal="LONG", entry_reason="CCI oversold")
        return EntrySignalOutput(signal="HOLD", entry_reason="")
```

Corrected: use direct attribute access:

In `strategy_params.py`:

```python
DEFAULT_PARAMS = {
    "entry_params": {
        "cci_threshold": 100,  # Make it explicit
        "printlog": False,
    },
    ...
}
```

In component:

```python
class EntryRule(BaseComponent):
    def generate_signal(self, bar_index: int) -> EntrySignalOutput:
        # RIGHT: direct access, fail loudly if missing
        cci_value = self.data[bar_index, 'cci']
        if cci_value < self.cci_threshold:  # No .get(), no fallback
            return EntrySignalOutput(signal="LONG", entry_reason="CCI oversold")
        return EntrySignalOutput(signal="HOLD", entry_reason="")
```
