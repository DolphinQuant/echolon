# VAL-006: Component method's return-type annotation is wrong

## Why this error fires

`validate_component_logging` in `echolon/strategy/validators/validate_component_logging.py`
raises VAL-006 when it inspects a component method's return-type annotation
and finds it doesn't match the expected BaseModel type (EntrySignalOutput,
ExitSignalOutput, RiskOutput, SizerOutput, etc.). At runtime, the strategy
would either fail Pydantic validation or emit the wrong schema to downstream
logging and analytics systems.

The context dict on the exception carries:

- `component` — the component name (e.g., "entry_rule")
- `method` — the method with the wrong annotation (e.g., "generate_signal")
- `expected_return` — the correct BaseModel type
- `actual_annotation` — what the method actually declares (or "missing")

## Fix

Update the method's return-type annotation to the expected BaseModel.
Annotations are critical for framework introspection and schema generation.

If the method has no annotation, add it. If it has the wrong annotation,
replace it with the correct one.

## Example

Component with wrong return annotation:

```python
from typing import Dict

class EntryRule(BaseComponent):
    def generate_signal(self, bar_index: int) -> Dict[str, Any]:
        # Wrong: returns Dict instead of EntrySignalOutput
        return {
            "signal": "LONG",
            "entry_reason": "CCI oversold",
        }
```

Corrected component:

```python
from echolon.core.signals import EntrySignalOutput

class EntryRule(BaseComponent):
    def generate_signal(self, bar_index: int) -> EntrySignalOutput:
        # Correct: returns the expected BaseModel
        return EntrySignalOutput(
            signal="LONG",
            entry_reason="CCI oversold",
        )
```
