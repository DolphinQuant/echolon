# BT-001: Strategy.on_bar() raised an exception

## Why this error fires

A strategy's entry/exit/risk/sizer component raised during a bar-level call. The exception was caught by the engine so the backtest could stop cleanly; the strategy code is the likely root cause.

## Fix

Open the file at the component that raised and reproduce with the context emitted alongside this error:

```
bar_index:       1247
trading_date:    2024-03-15
contract:        IF2403
position_size:   0
exception:       KeyError: 'atr_20'
```

The `exception` line tells you what Python saw. Most common culprits:

- `KeyError` on an indicator name — see [IND-001](IND-001.md) (casing) or [IND-002](IND-002.md) (undeclared).
- `IndexError` on a lookback — the indicator hasn't warmed up at `bar_index`; gate with `if bar_index < self.warmup: return HOLD`.
- `AttributeError` on `self.something` — a param from `strategy_params.py` is missing; see [PRM-002](PRM-002.md).

## Example

Triggering code (unguarded lookback before warmup):

```python
def evaluate(self, bar_index):
    # IndexError at bar_index < 20
    prev_atr = self.get_indicator("atr_20")[bar_index - 20]
    ...
```

Corrected version (explicit warmup guard):

```python
def evaluate(self, bar_index):
    if bar_index < 20:
        return EntrySignalOutput(signal="HOLD", strength=0.0, ...)
    prev_atr = self.get_indicator("atr_20")[bar_index - 20]
    ...
```
