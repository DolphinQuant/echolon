# BT-002: Backtest produced zero trades

## Why this error fires

The strategy ran through the configured period without firing a single entry. Common causes: entry conditions never met, filters block every signal, risk manager blocks every order.

## Fix

Read the diagnostics printed immediately above this error — they tell you which stage ate the signal:

```
bars_processed:          18240
entry_signals_generated:    72
entry_signals_blocked:      72
risk_blocks:                 0
```

Decision tree:

1. `entry_signals_generated == 0` — your entry condition never evaluated `True`. Check thresholds/comparators in `entry.py`.
2. `entry_signals_generated > 0` but `entry_signals_blocked == entry_signals_generated` — a regime filter or pre-trade check vetoed every signal. Inspect `filter_signal` / regime gates.
3. `risk_blocks > 0` matching the generated count — the risk manager (e.g., max-daily-loss or open-position cap) is blocking. Loosen constraints in `risk.py`.

## Example

Triggering code (entry condition that never fires because the RSI threshold is inverted for a long signal):

```python
class entry_rule(base_entry):
    def evaluate(self, bar_index):
        rsi = self.get_indicator("rsi_14")[bar_index]
        # Wrong: longs fire on high RSI — in practice rsi rarely exceeds 90
        if rsi > 90:
            return EntrySignalOutput(signal="LONG", strength=1.0, ...)
        return EntrySignalOutput(signal="HOLD", strength=0.0, ...)
```

Corrected version (standard oversold-reversal: long when RSI is low):

```python
class entry_rule(base_entry):
    def evaluate(self, bar_index):
        rsi = self.get_indicator("rsi_14")[bar_index]
        if rsi < 30:
            return EntrySignalOutput(signal="LONG", strength=1.0, ...)
        return EntrySignalOutput(signal="HOLD", strength=0.0, ...)
```
