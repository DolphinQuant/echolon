# CFG-001: end_date before start_date

## What
`BacktestConfig.end_date` is chronologically earlier than `start_date`.

## Why
A backtest cannot run backwards. The validator catches this before any data is loaded so you don't waste time on a run that can't produce bars.

## Fix
Swap the values or correct the typo:

```python
from echolon import BacktestConfig
from pathlib import Path

cfg = BacktestConfig(
    start_date="2020-01-01",   # earlier
    end_date="2023-12-31",     # later
    strategy_dir=Path("./my_strategy"),
    market_data_dir=Path("./data/market"),
    indicator_dir=Path("./data/indicators"),
    results_dir=Path("./results"),
)
```

If you also set `is_end_date` or `oos_start_date`, keep them inside `[start_date, end_date]`.

## Common Causes
- Typo in the year (`2202` vs `2022`)
- Copy-paste from a walk-forward split that used reversed windows
- Using `datetime.now()` as `start_date` with a stale hardcoded `end_date`
