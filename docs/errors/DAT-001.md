# DAT-001: Required OHLCV file not found

## What
Echolon expected a market data file at a specific path, but the file does not exist.

## Why
Before running a backtest, Echolon loads OHLCV data for the configured instrument, market, and frequency. The expected path is derived from the `TradingContext` (market, instrument, bar size) combined with `market_data_dir` from `BacktestConfig`.

Example resolution:
- `market="shfe"`, `instrument="cu"`, `bar_size="1d"` → `{market_data_dir}/shfe/cu/1d.parquet`

If that file isn't there, no backtest is possible.

## Fix
Either run the data pipeline that populates this directory, or point `market_data_dir` at an existing data store:

```python
from pathlib import Path
from echolon import BacktestConfig

cfg = BacktestConfig(
    start_date="2020-01-01",
    end_date="2023-12-31",
    strategy_dir=Path("./my_strategy"),
    market_data_dir=Path("/srv/echolon_data/market"),
    indicator_dir=Path("/srv/echolon_data/indicators"),
    results_dir=Path("./results"),
)
```

If only one instrument's file is missing, inspect the directory layout and ensure the pipeline has produced all instruments in the universe.

## Common Causes
- Data pipeline never run (or failed silently)
- Typo in instrument code (`cu` vs `CU` vs `copper`)
- Bar size mismatch (`1d` vs `daily` vs `1D`)
- `market_data_dir` points at the parent of the correct directory
