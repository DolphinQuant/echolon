# DAT-003: Conventional data file not found for instrument

## Why this error fires

Echolon resolves several per-instrument files under
`market_data_dir/{market}/{symbol}/...` by convention. The two sites that
raise DAT-003 today:

1. **`SessionAvailabilityLoader._load`** (trading-time bar-count lookup)
   expects `market_data_dir/{market}/{symbol}/session_availability.csv`.
2. **`shfe/contract_rules._load_main_contract_data`** (main-contract
   rollover lookup) expects `market_data_dir/{market}/{symbol}/main_contract.csv`.

When the conventional file is missing, the loader fails loud instead of
silently defaulting to synthetic or empty values — those silent fallbacks
caused hard-to-debug live-trading inconsistencies, so the policy is now
fail-loud.

Context dict:

- `path` — the full conventional path that was looked up
- `symbol` — the instrument symbol the loader was asked for

## Fix

Either (a) run the data pipeline to materialize the missing file at its
conventional path, or (b) pass an explicit `path_override` /
`market_data_dir` pointing at a populated tree. Option (a) is the normal
case; option (b) is for tests and ad-hoc runs.

## Example

Triggering code (asking for a symbol whose pipeline hasn't been run):

```python
from echolon.data.loaders.session_availability_loader import (
    SessionAvailabilityLoader,
)

# market_data_dir has "aluminum" but no "silver" tree yet
loader = SessionAvailabilityLoader(
    market="SHFE",
    instrument="silver",
    market_data_dir="workspace/data/market_data",
)
# Raises DAT-003 pointing at:
#   workspace/data/market_data/SHFE/silver/session_availability.csv
```

Corrected (run the pipeline first):

```bash
echolon data build --market SHFE --instrument silver
```

Then:

```python
loader = SessionAvailabilityLoader(
    market="SHFE",
    instrument="silver",
    market_data_dir="workspace/data/market_data",
)  # succeeds
```

Or, for tests, point at a fixture tree:

```python
loader = SessionAvailabilityLoader(
    market="SHFE",
    instrument="silver",
    market_data_dir="tests/fixtures/market_data",
)
```
