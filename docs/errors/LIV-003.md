# LIV-003: QMT async callback delivered an error

## Why this error fires

The miniQMT xtconstant callback for order/trade status indicated a failure outcome. The callback thread logs this but the main loop needs to translate it for the LLM agent monitoring the run.

## Fix

Translate the QMT status code and follow broker-specific remediation:
  seq_id:       {seq_id}
  qmt_status:   {qmt_status}
  echo_status:  {echo_status}
  raw:          {raw}
