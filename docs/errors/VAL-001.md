# VAL-001: Missing required field in component output

## What
A component method returned a Pydantic output object missing one or more required fields.

## Why
Echolon component contracts require specific fields for downstream processing:
- `regime` for regime-aware evaluation
- `type` for trade log categorization
- `entry_reason` / `exit_reason` / `risk_reason` / `sizing_reason` for debugging and evaluation

Missing any of these causes Pydantic validation to fail at runtime, before the bar is processed.

## Fix
Add all required fields to your output. For an entry signal:

```python
from echolon.quant_engine.core.interfaces.trading_interfaces import OrderIntent
from echolon.quant_engine.types import EntrySignalOutput

return EntrySignalOutput(
    signal="LONG",
    strength=1.0,
    type="entry_long",
    entry_reason="RSI oversold",
    intent=OrderIntent.ENTRY_LONG,
    regime=self.get_market_regime(),
)
```

For a HOLD signal, `intent` is optional but `regime`, `type`, and `entry_reason` are still required:

```python
return EntrySignalOutput(
    signal="HOLD",
    strength=0.0,
    type="hold",
    entry_reason="No setup",
    regime=self.get_market_regime(),
)
```

## Common Causes
- Copy-paste from an older Echolon version that had a smaller contract
- LLM agent hallucinating a simpler API
- Missing return statement in a conditional branch — Python returns `None`, validator fails downstream
