# LIV-002: Order rejected by broker

## Why this error fires

`_raise_order_rejected` (in
`echolon/live/platforms/miniqmt/qmt_client.py`) raises LIV-002 when the
broker returns a rejected-status response for a submitted order. The
exception captures the full submitted order side-by-side with the broker's
raw reply, so the agent reading logs can see the mismatch without
cross-referencing two systems.

Context dict:

- `contract` — the exchange contract code submitted (e.g. `"al2406"`)
- `direction` — `"BUY"` / `"SELL"` / `"BUY_TO_CLOSE"` etc.
- `price` — the limit price echolon submitted
- `size` — the order quantity (lots)
- `broker_status` — the native miniQMT status integer
  (see `_QMT_STATUS_TO_ECHO` in `qmt_client.py`; 57 = `REJECTED`)
- `broker_message` — the human-readable rejection reason the broker returned

**Note:** `_raise_order_rejected` is defined and tested but the on-response
wiring is a follow-up — as of now LIV-002 is primarily consumed by unit
tests and by external callers that invoke it directly after inspecting a
broker response.

## Fix

The rejection reason in `broker_message` is authoritative. Three common
SHFE categories:

1. **"不在合理价格范围"** (price outside the allowed band) — your limit
   price is too far from the last trade. Re-compute using the current
   reference price and widen the slippage band.
2. **"保证金不足"** (insufficient margin) — either trim size or check that
   capital was released from a prior exit before the entry was sent.
3. **"合约不存在"** (contract not listed) — the contract has rolled or
   expired; your strategy held onto a stale contract code. Refresh via
   `shfe.contract_rules.get_main_contract_for_date`.

## Example

Triggering code (submitting a price outside the allowed daily band):

```python
from echolon.live.platforms.miniqmt.qmt_client import MiniQMTClient

with MiniQMTClient(cfg) as client:
    # Last-traded aluminum price is ~20500; submitting 30000 is outside
    # the ±7% daily price band and will be rejected.
    client.submit_order(
        contract="al2406",
        direction="BUY",
        price=30000.0,
        size=1,
    )
# LIV-002 raised with broker_status=57, broker_message="价格超出涨跌停"
```

Corrected (clamp submission price near the reference):

```python
ref_price = client.get_latest_price("al2406")
limit = round(ref_price * 1.005, 0)   # half a percent of slippage
client.submit_order(
    contract="al2406",
    direction="BUY",
    price=limit,
    size=1,
)
```
