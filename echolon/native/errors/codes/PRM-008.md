# PRM-008: Param declared in params_to_optimize.json absent from optuna_search_space

## Why this error fires

A parameter declared in `params_to_optimize.json` (in a component's
`calculation`, `usage`, or `fixed` section) has no corresponding entry in
`strategy_params.py`'s `optuna_search_space` function — the two artifacts are out
of sync.

Typical cause: an EXPLOITATION value-tune regenerated or hand-edited
`strategy_params.py` and dropped the parameter, while `params_to_optimize.json`
(carried from the parent) still declares it. If the component code also reads the
dropped param, that's a bar-time `KeyError` (see [PRM-006](PRM-006.md)); PRM-008
catches the structural drift even before that.

Matching is by the **dict key** in `optuna_search_space`
(`entry_params["key"]`), checked against the **union** of all component dicts so a
shared/cross-component param (declared once in its owner's section) is never
falsely flagged.

This is a **validation finding** surfaced by `validate_params_sync` /
`validate_strategy_full`, not a runtime exception.

Finding fields: `param`, `section`, `sub`.

## How to fix

Either add the param back to `optuna_search_space` in `strategy_params.py`
(searched via `trial.suggest_*` or fixed via a plain assignment), or remove it
from `params_to_optimize.json` if it is intentionally gone.
