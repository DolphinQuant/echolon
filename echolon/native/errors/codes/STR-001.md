# STR-001: Strategy directory missing required file

## What
The strategy directory is missing one of the 7 required files.

## Why
Echolon's `StrategyLoader` loads components by convention — it expects each file to live at a known path. A missing file means a component can't be instantiated.

The 7 required files:
1. `strategy.py` — coordinator (`strategy_main` class)
2. `entry.py` — entry component (`entry_rule` class)
3. `exit.py` — exit component (`exit_rule` class)
4. `risk.py` — risk component (`risk_manager` class)
5. `sizer.py` — sizing component (`position_sizer` class)
6. `strategy_params.py` — `DEFAULT_PARAMS` dict
7. `strategy_indicator_list.json` — indicator declarations

## Fix
Easiest: scaffold a fresh strategy and copy your custom logic in:

```bash
echolon init my_strategy --template minimal
```

Or add the missing file by hand — the minimal template's file for that slot is a working starting point.

## Common Causes
- Hand-editing a directory and forgetting one file
- Merging branches that dropped a file
- Cloning a partial strategy from an example gist
