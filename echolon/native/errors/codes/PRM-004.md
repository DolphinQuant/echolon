# PRM-004: Defensive .get() on self.params

## Why this error fires

`validate_parameter_access` in `echolon/strategy/validators/parameter_access.py`
raises PRM-004 when it detects code using `self.params.get('x')` —
with or without a default. This pattern masks missing-parameter bugs:
without a default, a `KeyError` becomes `None` and silently corrupts
downstream logic; with a default, the strategy uses the fallback instead
of the value the user actually configured.

The framework contract is that every parameter declared in
`strategy_params.py`'s `entry_params` / `exit_params` / `risk_params` /
`sizer_params` is reliably present in `self.params` at runtime — the
strategy coordinator passes them as kwargs into each component, and
`BaseComponent.__init__` collects them into the `self._params` dict. So a
missing key indicates a real misconfiguration, not a runtime contingency
to defend against.

The context dict on the exception carries:

- `file` — the source file containing the .get() call
- `line` — the line number
- `call` — the full .get() expression

## Fix

Read each declared param into a typed instance attribute in `__init__`
using subscript access, then reference the attribute throughout the
component:

```python
class entry_rule(BaseComponent):
    def __init__(self, trading_engine, **params):
        super().__init__(trading_engine, **params)
        self.cci_period = self.params['cci_period']
        self.cci_threshold = self.params['cci_threshold']

    def generate_signal(self):
        cci = self.get_indicator(f"cci_{self.cci_period}")
        if cci < self.cci_threshold:
            return EntrySignalOutput(
                signal="LONG", strength=1.0, type="oversold",
                entry_reason="CCI oversold", intent=OrderIntent.ENTRY_LONG,
            )
        return EntrySignalOutput(
            signal="HOLD", strength=0.0, type="hold",
            entry_reason="",
        )
```

Subscript access (`self.params['x']`) raises `KeyError` if the parameter
is missing, which is what you want — load-time failure, not silent
fallback. Compare to the antipattern:

```python
# WRONG: silently substitutes 100 if cci_threshold isn't configured
cci_threshold = self.params.get('cci_threshold', 100)

# WRONG: silently becomes None if the key is missing
cci_threshold = self.params.get('cci_threshold')
```

If a parameter is genuinely optional, declare it in `strategy_params.py`
with a sensible default — that way it's always present in `self.params`
and the component can rely on `self.params['x']`.
