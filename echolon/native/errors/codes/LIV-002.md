# LIV-002: Order rejected by broker

## Why this error fires

LIV-002 is the canonical "broker declined a submitted order" code. It is
registered in `echolon.errors.ERROR_CATALOG` and meant to be raised from
the broker-response path (or from any caller that has just inspected a
rejection) via:

```python
from echolon.errors import raise_error

raise_error(
    "LIV-002",
    contract="al2406",
    direction="BUY",
    price=20000.0,
    size=1,
    broker_status=57,
    broker_message="价格超出涨跌停",
)
```

The catalog entry's `fix_template` lays out the context dict consumers
should populate.

Context dict:

- `contract` — the exchange contract code submitted (e.g. `"al2406"`)
- `direction` — `"BUY"` / `"SELL"` / `"BUY_TO_CLOSE"` etc.
- `price` — the limit price echolon submitted
- `size` — the order quantity (lots)
- `broker_status` — the native miniQMT status integer; see the canonical
  translation table at `echolon/live/config/qmt_constants.py`
  (`QMT_STATUS_MAP`); `57 == "REJECTED"`.
- `broker_message` — the human-readable rejection reason the broker returned

**Wiring status:** the live portfolio runner consumes the QMT order-status
integers directly when processing fills (see
`echolon/live/orchestrator/portfolio.py::_process_fills`, the `status == 57`
branch). The orchestrator does not currently raise LIV-002 — it logs the
rejection and continues the cycle. Future broker-response handlers that
want a typed exception should call `raise_error("LIV-002", ...)` with the
context dict above.

## Fix

The rejection reason in `broker_message` is authoritative. Three common
SHFE categories:

1. **"不在合理价格范围"** (price outside the allowed band) — your limit
   price is too far from the last trade. Re-compute using the current
   reference price and widen the slippage band.
2. **"保证金不足"** (insufficient margin) — either trim size or check that
   capital was released from a prior exit before the entry was sent.
3. **"合约不存在"** (contract not listed) — the contract has rolled or
   expired; your strategy held onto a stale contract code. Refresh via
   `shfe.contract_rules.get_main_contract_for_date`.
