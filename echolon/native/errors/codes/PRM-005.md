# PRM-005: Optimized trial parameters could not be resolved

## Why this error fires

`BacktestRunner.best_trial` must turn the recorded FLAT optuna params
(`selected_robust_trial.json`'s `params`) into the optimizer-exact NESTED
component vector (`{entry_params, exit_params, risk_params, sizer_params}`)
before it can run. It resolves this from two **authoritative** sources, in
order:

1. a sha-verified `resolved_params.json` companion sitting next to the trial
   file (written by the TrialSelector exporter), or
2. **on-demand replay** — loading the strategy dir's own `optuna_search_space`
   and replaying it with the recorded flat params (`resolve_via_replay`), which
   reproduces the exporter's vector byte-for-byte.

PRM-005 fires when **neither** works: there is no valid companion AND the
on-demand replay could not run (the dir's `strategy_params.py` failed to import
/ has no `optuna_search_space`, or a recorded suggested name is missing so the
replay can't reproduce the trial).

There is deliberately **no third fallback**. The historical strip-once
flat-name mapper was removed: it silently orphaned params whose canonical names
already carried a component prefix and dropped in-function shared copies, so a
backtest would run on *partially-correct* parameters and report a number nobody
could trust. A hard failure is strictly better than that silent flaw.

Context fields:

- `params_path` — the trial file that was loaded
- `strategy_code_dir` — the dir whose `optuna_search_space` replay was attempted

## Fix

Make the dir's params resolvable, then re-run. Either:

1. **Regenerate the artifacts** — re-run trial selection so the TrialSelector
   writes a fresh `selected_robust_trial.json` *and* its `resolved_params.json`
   companion (the exporter is wired into the WFA, single-split, and debug
   selection paths). This is the normal path for any freshly-optimized dir.
2. **Make on-demand replay work** — confirm `strategy_params.py` imports cleanly
   from `strategy_code_dir` and exposes `optuna_search_space`. Replay only needs
   the search-space function; it does not need a pre-written artifact, so a
   clean param module is sufficient for any pre-fix archived dir or deployed
   slot.

## Example

A deployed slot is re-backtested via `BacktestRunner.best_trial`. Its
`selected_robust_trial.json` exists but predates the resolved-params exporter,
so there is no companion. On-demand replay is attempted but
`strategy_params.py` raises on import (e.g. a relative import that does not load
under the current echolon), so replay returns None:

```
PRM-005: Optimized trial parameters could not be resolved to the component vector
  params_path:        .../strategy/al_s1/selected_robust_trial.json
  strategy_code_dir:  .../strategy/al_s1
```

Fix: ensure the vendored slot's `strategy_params.py` imports under the current
echolon (flat list + absolute `echolon.*` imports), or re-run selection to emit
the `resolved_params.json` companion.

## Common causes

- A pre-fix archived/deployed dir with no `resolved_params.json` whose
  `strategy_params.py` does not import cleanly (so replay can't run either).
- A hand-edited `selected_robust_trial.json` whose flat params no longer match
  the search space (a suggested name went missing → replay can't reproduce it).
- Pointing `best_trial` at a dir that is not a real strategy dir.
