# IND-008: Legacy section-keyed indicator-list format

## Why this error fires

`strategy_indicator_list.json` must be a **flat dict** mapping each indicator
name directly to its param spec:

```json
{"rsi": {"timeperiod": [10, 20]}, "atr": {"timeperiod": 14}, "market_regime": {}}
```

The catalog validator (`IndicatorCatalog.validate`, in
`echolon/indicators/catalog.py`) emits IND-008 when it finds a **deprecated
section-keyed** payload — one whose top level is a wrapper section rather than
indicator names:

```json
{
  "indicators_with_lookback":        {"rsi": {"timeperiod": [10, 20]}},
  "indicators_without_lookback":     {"obv": {}},
  "indicators_with_special_params":  {"market_regime": {}},
  "system_provided_indicators":      {}
}
```

Any of these four top-level keys triggers IND-008:
`indicators_with_lookback`, `indicators_without_lookback`,
`indicators_with_special_params`, `system_provided_indicators`.

This is a **validation finding** surfaced by the `validate_indicator_list`
MCP tool (`{"valid": false, "errors": [{code, field, message, suggestion}]}`),
not an exception raised mid-run. It is returned **alone** — the validator stops
before the per-name checks so you get one structural migration hint instead of a
junk IND-004 ("unknown indicator 'indicators_with_lookback'") per section.

Finding fields:

- `field` — the legacy section key(s) detected.
- `message` — the migration hint with a flat-dict example.

## How to fix

Flatten every section's entries up to the top level and drop the section
wrappers. The above becomes:

```json
{"rsi": {"timeperiod": [10, 20]}, "obv": {}, "market_regime": {}}
```

- Lookback indicators carry their sweep/param spec (`{"timeperiod": [10, 20]}`).
- Param-free indicators (incl. regime/session classifier columns like
  `market_regime`) use an empty `{}`.
- There is no separate "system-provided" section — declare every column the code
  reads (see [IND-002](IND-002.md)).
