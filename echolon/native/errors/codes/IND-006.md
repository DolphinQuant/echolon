# IND-006: Inverted [min, max] sweep range in strategy_indicator_list.json

## Why this error fires

The catalog range check (`IndicatorCatalog.validate`, in
`echolon/indicators/catalog.py`) inspects every parameter value in a
`strategy_indicator_list.json` payload. Any value written as a two-element
integer list `[min, max]` is treated as an Optuna sweep range. If `min > max`,
IND-006 is emitted: `suggest_int(low, high)` requires `low <= high`, so the
trial sampler would raise on the very first trial and no optimization could run.

This is a **validation finding** surfaced by the `validate_indicator_list`
MCP tool (`{"valid": false, "errors": [{code, field, message, suggestion}]}`),
not an exception raised mid-run.

Finding fields:

- `field` — the offending `"<indicator>.<param>"` (e.g. `rsi.timeperiod`)
- `message` — the inverted range, e.g. `[28, 14] — min (28) must be <= max (14)`
- `suggestion` — empty (the fix is mechanical: swap the bounds)

## Fix

Swap the two bounds in `strategy_indicator_list.json` so the smaller value
comes first. A sweep range is always `[low, high]` with `low <= high`. A single
fixed value should be a scalar, not a list.

## Example

Triggering payload (range written high-then-low):

```json
{
  "rsi": {"timeperiod": [28, 14]}
}
```

```python
from echolon.indicators import catalog
import json

errors = catalog.validate(json.loads('{"rsi": {"timeperiod": [28, 14]}}'))
# errors == [{
#   "code": "IND-006",
#   "field": "rsi.timeperiod",
#   "message": "Invalid range for 'rsi.timeperiod': [28, 14] — min (28) must be <= max (14).",
#   "suggestion": [],
# }]
```

Corrected payload (bounds swapped):

```json
{
  "rsi": {"timeperiod": [14, 28]}
}
```

If you meant a single fixed period rather than a sweep, use a scalar:

```json
{
  "rsi": {"timeperiod": 14}
}
```

## Common causes

- Copy-pasting a range and editing the numbers in the wrong order.
- Translating a "from 28 down to 14" mental model literally into `[28, 14]`.
- Mixing up a fixed value `[14]`/`14` with a range and padding it incorrectly.
