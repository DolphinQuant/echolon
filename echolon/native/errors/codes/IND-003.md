# IND-003: Indicator column is mostly NaN after warmup

## Why this error fires

After the indicator processor writes `strategy_indicators.csv`,
`_write_nan_warnings_sidecar` (in `echolon/indicators/engine/processor.py`)
scans each indicator column. Any column whose NaN ratio is ≥ 0.8 gets
recorded in a sidecar JSON file next to the CSV:

```
workspace/data/indicators/backtest/aluminum/strategy_indicators.csv
workspace/data/indicators/backtest/aluminum/strategy_indicators.csv.warnings.json
```

When a backtest later loads the CSV via
`backtest_data_loader.load_backtest_data`, the sidecar is read and each
flagged column surfaces as a `WARNING`:

```
[DATA_LOADER] IND-003: indicator 'atr_500' has 92.3% NaN (4620/5003 rows)
```

Note: IND-003 is emitted as a **warning**, not a raised exception. A
strategy that silently compares against the all-NaN column will produce
the zero-trade pattern (see [BT-002](BT-002.md)) — the warning is your
cue to fix the indicator spec before the run becomes a mystery.

Sidecar payload fields: `code`, `indicator`, `rows`, `nan_rows`,
`nan_ratio`.

## Fix

Either shorten the indicator period so warmup fits inside the available
history, or extend the backtest start date to allow warmup.

## Example

Triggering code (requesting a 500-period ATR against 5 years of daily data
— ≈1260 bars, so 500 bars of warmup is 40% of history and the remaining
60% still needs the *prior* 500 bars, producing mostly-NaN output for any
shorter backtest window):

```yaml
# strategy.yaml
indicators:
  - name: atr_500
    calculator: atr
    params:
      period: 500
```

When the backtest runs, sidecar is written with `nan_ratio: 0.923` for
`atr_500`, and the loader logs a WARNING at startup.

Corrected version (period sized to warmup budget):

```yaml
indicators:
  - name: atr_20
    calculator: atr
    params:
      period: 20
```

Or keep the long period and widen the historical window so warmup is
absorbed:

```python
# Start the backtest 500 bars earlier than the first signal you care about
run_backtest(start_date="2015-01-01", end_date="2024-12-31", ...)
```
