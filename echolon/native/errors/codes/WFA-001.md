# WFA-001: WFA pipeline produced zero valid trials across all windows

## Why this error fires

`WFARunner.run` in `echolon/backtest/wfa/runner.py` raises WFA-001 after the
per-window loop when **every** window finished its Optuna optimization with
zero successful trials. Previously this was a silent failure: the runner
logged "No windows completed successfully" and returned `{}`, which the host
app treated as a no-op empty result. Exiting zero with no artifacts is a
debuggability hazard — WFA-001 converts it into a loud, structured failure.

The context dict on the exception carries:

- `n_windows` — how many windows were configured
- `reason` — "All WFA windows produced zero valid trials"
- `suggestion` — pointer to per-window `trial_failure_summary.json` artifacts

## Fix

The optimizer has already written `trial_failure_summary.json` next to each
window's `optimization_trials.csv`. That JSON is the root-cause breadcrumb:

- `groups[0]` is the most common failure. `groups[0].error.error_type` +
  `.error_code` + `.message` + `.traceback` usually point at the exact bug.
- If the top group shows a `ValidationError` inside a component output
  (e.g. `EntrySignalOutput`), the strategy is emitting fields the schema
  rejects — fix the component or, if the fields are intentional diagnostics,
  ensure the schema's `extra='allow'` contract is respected (see the
  `test_schema_extras.py` regression test).
- If the top group shows a `KeyError` for an indicator, the strategy is
  reading from a column the indicator pipeline never produced.
- If the top group shows `ZeroDivisionError` / `OverflowError`, widen the
  parameter range or add a parameter-validity guard.

The artifact is structured JSON, so LLM debugger agents can parse
`groups[0].error` directly without regex-scraping the terminal.

## Example

A typical session ending in WFA-001 looks like this on stderr:

```
================================================================================
⚠  OPTIMIZATION FAILED — 200 of 200 trials failed (window 1)
================================================================================
GROUP 1 — 200 trials (100%)  [VAL-002]
  Error:   EchelonError
  Message: VAL-002: EntrySignalOutput missing required field entry_reason
  Trials:  first=0, last=199
  Context: component=entry, expected=entry_reason
  Docs:    https://github.com/dolphinquant/echolon/blob/master/echolon/native/errors/codes/VAL-002.md
  Exemplar traceback:
    ...
================================================================================
```

Plus the corresponding JSON at
`{paths.backtest_results_dir}/wfa_windows/window_1/trial_failure_summary.json`
(under the `echolon init` scaffold default, that resolves to
`workspace/backtest/wfa_windows/window_1/trial_failure_summary.json`).
