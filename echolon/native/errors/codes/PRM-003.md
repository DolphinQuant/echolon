# PRM-003: Hardcoded parameter value in component logic

## Why this error fires

`validate_parameter_access` in `echolon/strategy/validators/validate_parameter_access.py`
raises PRM-003 when it scans component code and finds a numeric or string
literal in a position where a parameter should be (e.g., in a condition,
threshold, or control value). The literal is effectively hardcoded and cannot
be tuned by Optuna. The optimizer will walk past that degree of freedom
blindly, missing potential improvements.

The context dict on the exception carries:

- `file` — the source file containing the literal
- `line` — the line number
- `literal` — the literal value (e.g., "30" or "'LONG'")
- `suggestion` — recommended parameter name to introduce

## Fix

Move the literal into `strategy_params.py` as a parameter, then reference it
in the component via `self.<param>` attribute access. This makes the value
tunable by Optuna.

Steps:

1. Add the parameter to the `strategy_params.py` dict (e.g., `"cci_threshold": 100`)
2. Reference it in the component as `self.cci_threshold` (not `self.params.get()`)
3. Re-run the validator to confirm the fix

## Example

Component with hardcoded literal:

```python
class EntryRule(BaseComponent):
    def generate_signal(self, bar_index: int) -> EntrySignalOutput:
        cci_value = self.data[bar_index, 'cci']
        if cci_value < 30:  # HARDCODED: 30 cannot be tuned
            return EntrySignalOutput(signal="LONG", entry_reason="CCI oversold")
        return EntrySignalOutput(signal="HOLD", entry_reason="")
```

Corrected: move 30 to params, reference via self:

In `strategy_params.py`:

```python
DEFAULT_PARAMS = {
    "entry_params": {
        "cci_threshold": 30,  # Now tunable
        "printlog": False,
    },
    ...
}
```

In component:

```python
class EntryRule(BaseComponent):
    def generate_signal(self, bar_index: int) -> EntrySignalOutput:
        cci_value = self.data[bar_index, 'cci']
        if cci_value < self.cci_threshold:  # Now references parameter
            return EntrySignalOutput(signal="LONG", entry_reason="CCI oversold")
        return EntrySignalOutput(signal="HOLD", entry_reason="")
```
