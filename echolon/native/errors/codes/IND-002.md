# IND-002: Indicator name has no calculation mapping

## What
Your `strategy_indicator_list.json` declares an indicator name that the indicator pipeline can't resolve to a calculation function — e.g. a typo (`"rsii"` instead of `"rsi"`), a renamed-but-not-yet-shipped name, or an indicator that's not in echolon's catalog.

## Why
The indicator processor (`echolon/indicators/engine/processor.py::_resolve_function`) looks up each declared name in the `INDICATOR_MAPPING` registry. If the name has no mapping, it raises IND-002 immediately rather than emitting an all-NaN column you couldn't debug downstream.

This is a JSON-side validation, not a strategy-code-side one. **Strategy code calling `get_indicator("foo")` for a name not declared in JSON does NOT trigger IND-002** — it surfaces as a runtime `KeyError` from `market_data.get_indicator` (typically wrapped into BT-001 if it happens during a backtest bar). Pre-flight validators today do not catch the strategy-code-vs-JSON mismatch; treat that as a separate failure mode.

## Fix
Verify the indicator name exists in echolon's catalog before declaring it in JSON:

```python
from echolon.indicators import catalog
catalog.info("rsi")        # IndicatorInfo(...)
catalog.info("rsii")       # None  ← would trigger IND-002
catalog.suggest_similar("rsii")  # ['rsi', ...]
```

Or via MCP: `validate_indicator_list(payload_json)` round-trips your `strategy_indicator_list.json` against the catalog before the processor ever runs.

The flat-dict format:

```json
{
  "atr": {"timeperiod": 14},
  "rsi": {"timeperiod": 14},
  "highest_high": {"timeperiod": [10, 50]},
  "market_regime": {}
}
```

Two-element integer lists like `[10, 50]` produce one column per period (`highest_high_10` … `highest_high_50`). Every period your strategy code references must fall inside the declared range, otherwise `get_indicator` raises `KeyError` at backtest time (not IND-002).

To discover valid names + accepted params:

- MCP: `list_indicators(has_lookback=...)`, `indicator_info(name)`, `suggest_similar(name)`
- Python: `echolon.indicators.catalog.list_all()`, `catalog.info(name)`
- CLI: `echolon indicators list`

## Common Causes
- Typo when authoring the JSON (`"rsii"`, `"adsx"`, `"hihest_high"`)
- Renaming an indicator in the catalog but forgetting to update existing strategy JSONs
- Carrying over an indicator name from another framework (TradingView, backtrader) that echolon doesn't ship
