# IND-001: Indicator name casing mismatch between code and JSON

## What
Your strategy code calls `self.get_indicator("ATR_14")` (uppercase), but the pre-computed indicator column is named `atr_14` (lowercase).

## Why
Echolon pre-computes indicators into a DataFrame where all column names are lowercase. `get_indicator` does a literal column lookup — no case folding, no fuzzy matching. An uppercase call raises `KeyError` from `market_data.get_indicator` (`echolon/backtest/engine/backtrader_engine.py:196`). Catching that `KeyError` and substituting `None` masks the bug and lets a misnamed lookup quietly turn into a strategy that never trades — don't do it.

## Fix
Lowercase every indicator name in code:

```python
# wrong
atr = self.get_indicator("ATR_14")
rsi = self.get_indicator(f"RSI_{self.rsi_period}")

# right
atr = self.get_indicator("atr_14")
rsi = self.get_indicator(f"rsi_{self.rsi_period}")
```

`strategy_indicator_list.json` uses lowercase canonical names: `{"atr": {"timeperiod": 14}}`. Names are case-insensitive on input — `IndicatorList` lowercases for you — but the resulting column names are always lowercase, so write your code in lowercase too.

## Common Causes
- Auto-complete from pandas / NumPy docs that use uppercase
- LLM agent following TradingView / Pine Script conventions
- Copy-pasting an indicator-family name from external docs without lowercasing
