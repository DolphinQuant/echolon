# BT-002: Backtest produced zero trades

## Why this error fires

`BacktestRunner._log_summary` calls `_assert_trades_produced` once after the
run completes. When `results.total_trades == 0`, it logs a
`ZERO TRADES WARNING` line (via `log_zero_trades_warning`) and then raises
BT-002. Common causes: entry conditions never evaluate True, a filter blocks
every signal, or the risk manager refuses every order.

**Scope note:** This check fires once per backtest run, not per Optuna trial.
When optimizing, a trial that produces zero trades still raises BT-002 from
inside the trial's runner; Optuna will catch the exception and record the
trial as failed.

## Fix

Scroll up and read the `ZERO TRADES WARNING` line logged immediately before
this error:

```
[BACKTEST] Backtest | ZERO TRADES WARNING | bars=18240, entry_signals=0, signals_blocked=0, risk_blocks=0 | HINT: No entry signals generated - check entry conditions
```

**Current limitation:** the `entry_signals`, `signals_blocked`, and
`risk_blocks` counters are not yet populated by strategies — they are always
reported as 0. Rely on `bars` + the HINT line, then walk your strategy by
hand to find which stage consumed every bar.

Debug checklist:

1. **Inspect `entry_rule.generate_signal`** — does the comparison ever evaluate
   True for any bar? Print the indicator value at a random bar and compare
   against your threshold.
2. **Inspect filter and regime gates** — if you vetoed by regime, is the
   calendar-loaded regime column actually populated for the backtest window?
3. **Inspect `risk_rule` / `sizer_rule`** — does the risk module clamp every
   candidate size to 0?

## Example

Triggering code (RSI threshold inverted for a long signal — `rsi > 90`
almost never evaluates True):

```python
class entry_rule(base_entry):
    def generate_signal(self, bar_index):
        rsi = self.get_indicator("rsi_14")[bar_index]
        if rsi > 90:
            return EntrySignalOutput(signal="LONG", strength=1.0)
        return EntrySignalOutput(signal="HOLD", strength=0.0)
```

Corrected version (oversold-reversal: long when RSI is low):

```python
class entry_rule(base_entry):
    def generate_signal(self, bar_index):
        rsi = self.get_indicator("rsi_14")[bar_index]
        if rsi < 30:
            return EntrySignalOutput(signal="LONG", strength=1.0)
        return EntrySignalOutput(signal="HOLD", strength=0.0)
```
