# PRM-006: Parameter read via self.params['X'] but absent from DEFAULT_PARAMS

## Why this error fires

A strategy component reads a parameter by subscript:

```python
self.atr_period = self.params['atr_14_period']   # in entry/exit/risk/sizer __init__
```

If `'atr_14_period'` is not present in `DEFAULT_PARAMS` (declared in
`strategy_params.py`), this raises `KeyError` **at bar-time** — minutes into the
backtest, not at load. PRM-006 lifts that failure to **preflight**.

The validator (`validate_parameter_access`, in
`echolon/strategy/validators/parameter_access.py`):
- **AST-scans** every component file for `self.params['<string-constant>']`
  reads (complete coverage — all code paths, not just the bar-0 path), and
- cross-checks each key against the **runtime-loaded** union of all
  `DEFAULT_PARAMS` sections. (`DEFAULT_PARAMS` is composed by
  `framework.compose_default_strategy()` plus cross-section copies, so it is
  loaded via `StrategyLoader` — it cannot be parsed statically.)

A key absent from every section → PRM-006.

This is a **validation finding** surfaced by `validate_parameter_access` /
`validate_strategy_full` (`{"findings": [{code, message, context}]}`), not an
exception raised mid-run.

Finding fields:

- `param` — the missing parameter name.
- `file`, `line` — where it is read.

## Common cause

An EXPLOITATION value-tune regenerated `strategy_params.py` with only the
*tuned* subset of params and dropped a **fixed** param the component code still
reads (e.g. `min_pathway_confidence`, `initial_capital`). The code↔params
contract drifted.

## How to fix

Either:
- **Declare the param** in `strategy_params.py` (add it to the relevant
  `*_params` section), or
- **Carry it forward** in the value-tune so the regenerated params keep the
  parent's fixed entries, or
- **Remove the read** if the param is genuinely no longer used.
