# STR-002: Required class not found in file

## What
A strategy file exists, but it does not export the class name `StrategyLoader` expects.

## Why
Echolon loads components by exact class name. The mapping is strict:

| File | Expected class |
|------|----------------|
| `strategy.py` | `strategy_main` |
| `entry.py` | `entry_rule` |
| `exit.py` | `exit_rule` |
| `risk.py` | `risk_manager` |
| `sizer.py` | `position_sizer` |

No auto-discovery, no "first class in the file" fallback — this is intentional, so the loader never guesses wrong.

## Fix
Rename your class to match. For example, in `entry.py`:

```python
# wrong
class MyEntrySignal(BaseComponent):
    ...

# right
class entry_rule(BaseComponent):
    ...
```

If you want a human-friendly name internally, keep it as an alias:

```python
class entry_rule(BaseComponent):
    """Breakout entry — was MyEntrySignal."""
    ...
```

## Common Causes
- Following Python convention (`PascalCase` classes) instead of Echolon convention (`snake_case` component classes)
- LLM regenerating the file and inventing a class name
- Importing a class from a helper module without re-exporting it under the right name
