# VAL-002: Invalid enum value in signal field

## What
A component returned a `signal` value that is not one of the allowed enum literals.

## Why
The `signal` field is typed as `Literal['LONG', 'SHORT', 'HOLD']`. Values like `'BUY'`, `'SELL'`, `'long'` (lowercase), or `1` / `-1` / `0` are not accepted.

This strict typing prevents a class of silent bugs where downstream code misinterprets direction.

## Fix
Use exactly one of `'LONG'`, `'SHORT'`, `'HOLD'`:

```python
return EntrySignalOutput(
    signal="LONG",           # not 'BUY', not 'long'
    strength=1.0,
    type="entry_long",
    entry_reason="Breakout",
    intent=OrderIntent.ENTRY_LONG,
    regime=self.get_market_regime(),
)
```

For exits, `signal_direction` on `SizerOutput` takes the same three literals.

## Common Causes
- Coming from another framework (backtrader: `'buy'`; quantopian: `+1` / `-1`)
- LLM substituting intuitive English words
- Typo or wrong case (`'Long'`, `'LONG '` with trailing space)
