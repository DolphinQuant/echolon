# LIV-001: Broker connection unavailable

## Why this error fires

`_raise_broker_unavailable` (in
`echolon/live/platforms/miniqmt/qmt_client.py`) is invoked when the
miniQMT client cannot connect at all, or loses connection during a session.
It is wired into `MiniQMTClient.__enter__`: if `connect()` returns False
after all retries, LIV-001 is raised rather than returning a half-initialized
client that would silently reject every subsequent order.

Context dict:

- `platform` — always `"miniqmt"` from this call site
- `account_id` — the configured QMT account id
- `error` — a short message pointing at the miniQMT error code / traceback

## Fix

Inspect the log lines immediately above the exception: miniQMT emits its
native error code and any login / session diagnostic on connection failure.
Typical root causes, in order of frequency:

1. **miniQMT desktop client not running** — launch it and log in before
   the deploy starts.
2. **Wrong `account_id` or mismatched account type** — verify against the
   logged-in client.
3. **Network partition between runner and miniQMT host** — try a plain
   TCP check to the configured host/port.
4. **QMT session expired** — re-authenticate in the desktop client.

## Example

Triggering code (deploy started before miniQMT is running):

```python
from echolon.live.platforms.miniqmt.qmt_client import MiniQMTClient
from echolon.live.config.deploy_config import QMTAccountConfig

cfg = QMTAccountConfig(account_id="8880000123", ...)
with MiniQMTClient(cfg) as client:   # raises LIV-001: connect() returned False
    ...
```

Corrected (make sure miniQMT is up before running):

```bash
# 1. Start miniQMT desktop and log in
# 2. Then run the deploy
echolon deploy start --config workspace/deploy/prod.yaml
```

Or, for automated environments, add a connectivity precheck:

```python
import socket

def _qmt_reachable(host: str, port: int, timeout: float = 2.0) -> bool:
    try:
        with socket.create_connection((host, port), timeout=timeout):
            return True
    except OSError:
        return False

if not _qmt_reachable(cfg.host, cfg.port):
    raise SystemExit("miniQMT not reachable — start the client first.")

with MiniQMTClient(cfg) as client:
    ...
```
