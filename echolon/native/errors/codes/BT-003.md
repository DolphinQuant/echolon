# BT-003: Optuna trial violated a hard constraint

## Why this error fires

`_raise_constraint_violation` in `echolon/backtest/optimization/optuna_study.py`
raises BT-003 when a trial's parameter set produces metrics outside the
viability bounds declared on `RegimeOptimizerConfig`
(`min_volatile_pct`, `min_ranging_pct`, `min_trending_pct`,
`max_single_regime_pct`, `min_return_separation`, etc.). The trial's score
is clamped to 0.0 so it cannot be selected as best.

The context dict on the exception carries:

- `trial_number` — the rejected trial's ordinal
- `constraint` — which threshold tripped (e.g. `"min_volatile_pct"`)
- `required` — the threshold value
- `actual` — what the trial actually produced
- `params` — the full parameter dict that Optuna sampled

## Fix

Widen whichever bound tripped, or widen the parameter range that keeps
producing degenerate regimes. Each rejection carries enough context to know
which knob is the problem — look at `constraint` + `actual` first.

If *every* trial is being rejected (not just some), you will also see
[IND-004](IND-004.md) at the end of the run. At that point the fix is the
same but more urgent: the optimizer has nothing to return.

## Example

Triggering config (hard constraint set so tight that no realistic regime
split can satisfy it):

```python
from echolon.indicators.optimization.interday_regime_optimizer import (
    RegimeOptimizerConfig,
    optimize_regime_params,
)

cfg = RegimeOptimizerConfig(
    n_trials=100,
    min_volatile_pct=20.0,     # demand 20% volatile regime — unrealistic
    min_trending_pct=20.0,     # demand 20% in each direction — unrealistic
    max_single_regime_pct=40.0,  # leaves almost no slack
)
# Most trials will trip min_trending_pct -> BT-003 raised, score=0.0
optimize_regime_params(ctx, data_dir=..., config=cfg)
```

Corrected config (realistic bounds, matching the library defaults):

```python
cfg = RegimeOptimizerConfig(
    n_trials=100,
    min_volatile_pct=1.0,
    min_ranging_pct=5.0,
    min_trending_pct=2.0,
    max_single_regime_pct=85.0,
)
optimize_regime_params(ctx, data_dir=..., config=cfg)
```
