# DAT-002: State file is corrupt or unreadable JSON

## Why this error fires

`TradingSlot._load_state_file` (in `echolon/live/trading_slot.py`) reads
`strategy_state.json` at the start of every live session so the slot can
resume position counts, cycle markers, and post-fill bookkeeping. If the
file does **not** exist the loader returns `{}` (cold start is explicitly
allowed). But if the file exists and `json.load` raises `JSONDecodeError`,
DAT-002 is raised rather than silently defaulting to an empty state — a
corrupt file otherwise means the slot would re-open positions it already
filled and re-fire one-shot signals.

Context dict:

- `path` — absolute path to the offending file
- `error` — the JSONDecodeError string (line, column, reason)

## Fix

Open the state file and either repair the JSON or delete the file to force
a clean cold start. Deletion is safe only if you are sure the slot has no
open position — otherwise the live controller will not know to close what
the broker already holds.

## Example

Triggering code (an external script truncated the file mid-write):

```python
# some_external_tool.py
# Bug: opens the file for write then crashes before flushing, leaving
# "strategy_state.json" with incomplete JSON on disk:
with open("workspace/live/SLOT-A/strategy_state.json", "w") as f:
    f.write('{"position": 1, "last_fill":')  # process killed here
```

Next live session:

```python
# Raises DAT-002 before any trading happens
slot = TradingSlot(slot_config, deploy_data_dir="workspace/live")
slot.initialize(present_date=today)
```

Corrected recovery (one of the following):

```python
# Option 1: repair by hand — valid JSON with the last known good state
#   strategy_state.json should be a complete JSON object.

# Option 2: cold-start (ONLY if you verified no open position at broker)
import os
os.remove("workspace/live/SLOT-A/strategy_state.json")
```
