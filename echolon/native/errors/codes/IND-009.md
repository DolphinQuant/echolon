# IND-009: curve_carry indicator requested from the single-frame compute path

## Why this error fires

`echolon.indicators.run.compute_indicators_from_frame` is the injectable-frame
entry point: it takes one caller-provided continuous OHLCV `DataFrame` (e.g. a
block-bootstrap / synthetic-data series) and runs the same per-bar indicator
computations the standard pipeline (`run_indicator_calculation`) applies to its
stitched main-contract series — without touching contract files or the roll
table.

`curve_carry`-kind indicators (`carry_front_back`, `carry_z_3m`, and the other
forward-curve carry signals) are the one indicator family the standard pipeline
does **not** compute per-contract. They are built once from the full
multi-contract forward curve — every listed contract's term structure on a
given date — by
`echolon.indicators.calculators.interday.carry.series_builder.build_carry_indicator_frame`,
which reads the raw multi-contract curve file directly. A single continuous
OHLCV frame carries only one price series per bar; it has no other contracts'
quotes to build a curve from. Silently dropping the columns or approximating
them from the single series would hand a parity-bound caller (e.g. bootstrap
resampling that assumes indicator parity with the archived backtest) a
misleading number instead of an honest failure, so `compute_indicators_from_frame`
raises IND-009 as soon as it sees a `curve_carry` name in `indicator_list`
— before any computation runs.

Context dict:

- `indicators` — the offending `curve_carry` name(s) found in `indicator_list`.

## Fix

Pick one:

1. **Drop the curve_carry names** from the `indicator_list` passed to
   `compute_indicators_from_frame` — compute everything else from the frame,
   and treat carry as out of scope for that call.
2. **Compute carry separately** from the real forward curve via
   `build_carry_indicator_frame(...)` and merge the columns onto your frame's
   output yourself, the same way
   `IndicatorProcessor._merge_curve_carry_indicators` does for the standard
   pipeline.

## Example

Triggering call:

```python
from echolon.indicators.run import compute_indicators_from_frame

compute_indicators_from_frame(
    ohlcv=continuous_ohlcv_df,
    indicator_list={"atr": {"timeperiod": 14}, "carry_front_back": {}},
    ctx=ctx,
)
# raises IND-009
# context: indicators=['carry_front_back']
```

Corrected (carry dropped; everything else still computed from the frame):

```python
compute_indicators_from_frame(
    ohlcv=continuous_ohlcv_df,
    indicator_list={"atr": {"timeperiod": 14}},
    ctx=ctx,
)
```
